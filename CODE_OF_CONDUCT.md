# Code of Conduct

## Our Pledge

We are committed to making CTEM Leader Lab a respectful, inclusive, and useful project for security leaders, practitioners, educators, and contributors.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent while still being clear about risks and mistakes.
- Keep discussions focused on CTEM implementation, security leadership, and safe learning.
- Use inclusive language.
- Credit sources and prior work when contributing content.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Discriminatory language or conduct.
- Publishing private information without consent.
- Encouraging unauthorized testing, exploitation, or unsafe security activity.
- Repeatedly derailing technical or project discussions.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants who violate this code of conduct. Serious incidents should be reported through the security or maintainer contact path documented in `SECURITY.md`.

## Scope

This code applies in project spaces including GitHub issues, pull requests, discussions, documentation, community calls, and related channels.
